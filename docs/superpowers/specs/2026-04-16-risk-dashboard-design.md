# 综合风险大屏 — 设计文档

**项目：** 基于可视化技术的铁路明桥面步行板安全隐患预警系统
**日期：** 2026-04-16
**定位：** 原型演示系统（面向领导/客户投屏展示）

---

## 概述

在现有系统基础上新增三大核心能力：综合风险评分、预警预测时间线、风险热力大屏页面。目标是打造一个视觉冲击力强的演示大屏，让决策者一眼掌握全局桥梁安全态势。

---

## 模块一：综合风险评分模型

### 目标

给每座桥一个 0-100 的健康分数，一眼判断桥梁状况。

### 评分维度

| 因子 | 权重 | 计算方式 |
|------|------|----------|
| 损坏率 | 30% | 高危板（severe_damage + fracture_risk）占全部板的比例 |
| 趋势恶化度 | 25% | 近30天损坏率变化趋势（基于 BoardStatusSnapshot 数据） |
| 高危集中度 | 20% | 高危板是否集中在某个跨度（跨度级别聚集度计算） |
| 检修时效 | 15% | 距上次完成检修任务的天数，越久扣分越多 |
| 覆盖完整度 | 10% | 已巡检板（有检查记录的板）占全部板的比例 |

### 评分等级

| 分数 | 颜色 | 状态 |
|------|------|------|
| 90-100 | 绿色 | 安全 |
| 70-89 | 蓝色 | 良好 |
| 50-69 | 黄色 | 需关注 |
| 30-49 | 橙色 | 较高风险 |
| 0-29 | 红色 | 高危 |

### 呈现位置

- 桥梁列表/选择器中每个桥旁边显示分数和颜色标识
- 2D 网格视图顶部显示当前桥的综合评分卡片
- 仪表盘页面中显示所有桥的评分排名

### 数据来源

全部来自已有的 `WalkingBoard`、`BridgeSpan`、`BoardStatusSnapshot`、`InspectionTask` 数据，无需新增数据表。

### 新增文件

- `src/app/api/stats/risk-score/route.ts` — 风险评分计算 API
- `src/lib/risk-score.ts` — 评分算法逻辑

## 模块二：预警预测时间线

### 目标

基于历史快照数据，预测未来30天各桥的损坏趋势，标注预计预警节点。

### 预测算法

1. 从 `BoardStatusSnapshot` 提取每天的损坏率数据
2. 对近30天数据做7日移动平均平滑处理
3. 基于平滑后数据做线性回归
4. 外推30天得到预测曲线
5. 预测曲线与告警阈值线（损坏率15%）的交叉点即为预计预警日

### 可视化设计

- **X轴：** 时间线（过去30天 + 未来30天预测区域）
- **Y轴：** 损坏率百分比
- **实线：** 历史实际数据
- **虚线：** 预测趋势
- **红色水平线：** 告警阈值
- **红色标记点：** 预计触发预警的日期节点
- 预测区域用半透明背景与历史区域区分

### 交互设计

- 在趋势分析页面（`TrendAnalysis.tsx`）新增"预测模式"切换按钮
- 支持选择不同桥梁对比预测趋势
- 预测标记点鼠标悬停显示具体信息

### 数据来源

已有的 `BoardStatusSnapshot` 表，无需新增数据表。

### 新增文件

- `src/app/api/stats/prediction/route.ts` — 预测数据 API
- `src/lib/prediction.ts` — 预测算法逻辑

## 模块三：风险热力大屏

### 目标

全新全屏大屏页面，深色科技风格，适合投屏演示。

### 路由

新增 `/screen` 页面（`src/app/screen/page.tsx`）

### 页面布局

```
┌─────────────────────────────────────────────┐
│            系统标题 + 当前时间               │
├──────────────────────┬──────────────────────┤
│                      │   右上：实时预警滚动  │
│   左侧：桥梁地图     │   （最新预警信息流）  │
│   （风险热力标记）    ├──────────────────────┤
│                      │   右中：风险评分排名  │
│                      │   （Top5 高危桥梁）   │
├──────────────────────┼──────────────────────┤
│   左下：整体统计      │   右下：预测趋势     │
│   （总桥数/总板数/    │   （全局损坏率预测）  │
│    高危数/预警数）    │                      │
└──────────────────────┴──────────────────────┘
```

### 各区域详情

**1. 顶部标题栏**
- 系统全称 + 科幻风格装饰线
- 实时时钟（秒级更新）
- 整体状态指示灯（全绿 / 有黄 / 有红）

**2. 左侧：桥梁地图**
- 复用 `/map` 页面的 SVG 地图能力
- 增强：每座桥按风险评分着色（绿/黄/橙/红）
- 高危桥标记添加脉冲呼吸动画效果

**3. 右上：实时预警滚动**
- 滚动显示最近预警记录（从 `AlertRecord` 获取）
- 每条显示：桥梁名称、预警类型、严重程度、时间
- 严重预警红色背景闪烁

**4. 右中：风险评分排名**
- 水平条形图展示 Top 5 风险最高的桥
- 颜色对应评分等级，带分数标注

**5. 左下：整体统计卡片**
- 4 个数字卡片：桥梁总数、步道板总数、高危板数、活跃预警数
- 数字动态翻牌动画效果

**6. 右下：全局预测趋势**
- 所有桥的加权平均损坏率预测曲线
- 复用模块二的预测算法能力

### 视觉风格

- 深色科技风，与现有夜间主题（霓虹青色 `#00f0ff`）一脉相承
- 全屏布局，无导航栏、无侧边栏
- 背景使用深蓝渐变 + 网格线装饰

### 数据刷新

每 30 秒自动轮询更新所有数据。

### 新增文件

- `src/app/screen/page.tsx` — 大屏页面
- `src/app/screen/layout.tsx` — 全屏布局（隐藏导航）
- `src/components/screen/ScreenHeader.tsx` — 标题栏
- `src/components/screen/ScreenMap.tsx` — 大屏地图
- `src/components/screen/ScreenAlertFeed.tsx` — 预警滚动
- `src/components/screen/ScreenRiskRanking.tsx` — 风险排名
- `src/components/screen/ScreenStats.tsx` — 统计卡片
- `src/components/screen/ScreenPrediction.tsx` — 预测趋势图
- `src/hooks/useScreenData.ts` — 大屏数据聚合 hook

### 复用的已有 API

- `/api/summary` — 整体统计
- `/api/alerts` — 预警记录
- `/api/stats/risk-score` — 风险评分（模块一新增）
- `/api/stats/prediction` — 预测数据（模块二新增）

## 实现优先级

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | 综合风险评分模型 | 其他模块的数据基础 |
| P1 | 预警预测时间线 | 集成到现有趋势分析页 |
| P2 | 风险热力大屏 | 最终投屏展示页面，依赖前两个模块 |

---

## 不在范围内

- 不引入新的外部 AI/ML 服务，预测算法使用纯数学方法
- 不新增数据库表，全部基于现有数据
- 不修改现有的告警引擎核心逻辑
- 不涉及后端部署和性能优化
